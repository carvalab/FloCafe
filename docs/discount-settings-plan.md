# Discount Settings Implementation Plan

Generated by CEO Review (HOLD SCOPE) — 2026-07-05

## Problem

`discount_max_amount` defaults to 100 with no UI to change it. A restaurant owner trying to give a ₹150 loyalty discount gets rejected with "discount_value exceeds maximum amount of 100." The setting exists in the DB but is invisible.

Additionally, `discount_mode` and `discount_requires_approval` are inserted in `db.ts:535-536` but never wired up — dead settings.

## Implementation Tasks

### T1 (P1, CC: ~5min) — Backend: Add discount settings route

Add `GET /settings/discount` and `PUT /settings/discount` to `main/routes/settings.ts`.

**GET response shape:**
```json
{
  "discount_max_percentage": 50,
  "discount_max_amount": 100,
  "discount_mode": "both",
  "discount_requires_approval": false
}
```

**PUT validation:**
- `discount_max_percentage`: parseFloat, range 0–100 (0 = no limit for %)
- `discount_max_amount`: parseFloat, range 0–999999 (0 = no limit for flat)
- `discount_mode`: must be 'percentage', 'flat', or 'both'
- `discount_requires_approval`: boolean (coerce string → bool)
- Reject with 400 on invalid input

**Pattern:** Follow `GET/PUT /settings/loyalty` exactly (lines 140-195).

**Files:** `main/routes/settings.ts`

### T3 (P1, CC: ~5min) — Backend: Wire up discount_requires_approval

When `discount_requires_approval` is true, discounts require a valid manager/owner PIN.

**Approach:** The existing `verifyPin` utility is already used in order status changes (`orders.ts:477-484`). Add similar check to discount endpoints.

**In `bills.ts` (discount endpoint) and `orders.ts` (discount endpoints):**

```typescript
const requiresApproval = getSettingValue('discount_requires_approval') === 'true';
if (requiresApproval && discount_value > 0) {
  const { override_pin } = req.body;
  if (!override_pin) {
    return res.status(403).json({ error: 'Manager PIN required for discounts', requiresApproval: true });
  }
  // Verify PIN (same pattern as orders.ts:478-484)
  const user = db.prepare("SELECT * FROM users WHERE pin_hash IS NOT NULL AND role IN ('owner', 'manager')")
    .all()
    .find((u: any) => verifyPin(u.pin_hash, override_pin));
  if (!user) {
    return res.status(403).json({ error: 'Invalid manager PIN' });
  }
}
```

**Files:** `main/routes/bills.ts`, `main/routes/orders.ts` (×2)
